# Dockhand

Docker-контейнер [Dockhand](https://github.com/typehero/dockhand) с управлением через Make и зашифрованным резервным копированием.

## Требования

- Docker и Docker Compose
- Make
- OpenSSL (для шифрования резервных копий)

## Установка

```bash
# 1. Скопировать конфигурацию
cp .env.example .env

# 2. Создать Docker-сеть (один раз)
docker network create dockhand_network

# 3. Запустить контейнер
make run
```

Веб-интерфейс доступен по адресу: http://localhost:3000

## Конфигурация

Заполните `.env`:

| Переменная                  | Описание                                       |
|-----------------------------|------------------------------------------------|
| `HOME_DIR`                  | Домашняя директория для монтирования в контейнер |
| `DOCKHAND_DATA`             | Путь к данным контейнера                       |
| `DOCKHAND_BACKUPS`          | Путь для хранения резервных копий              |
| `DOCKHAND_BACKUP_PASSWORD`  | Пароль для шифрования резервных копий          |

## Команды

```bash
make run        # Запустить контейнер
make stop       # Остановить контейнер
make restart    # Перезапустить контейнер
make backup     # Создать зашифрованный резервную копию директории data
make restore    # Восстановить из последней резервной копии
make help       # Список всех доступных команд
```

## Резервное копирование

Резервные копии шифруются с использованием AES-256-CBC (PBKDF2, 600 000 итераций) и сохраняются в `DOCKHAND_BACKUPS` с контрольными суммами SHA256.

```bash
make backup   # Остановить → создать резервную копию → запустить
make restore  # Остановить → восстановить последний резервную копию → запустить
```

Ручное управление через скрипт:

```bash
./scripts/backup.sh backup                           # Создать резервную копию
./scripts/backup.sh restore                          # Восстановить последний
./scripts/backup.sh restore /path/to/archive.tar.gz.enc  # Из конкретного файла
```
